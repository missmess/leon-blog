# Vue2# 架构

闲来无事，突发奇想，来造个轮子吧：如何在React中，仿照Vue的框架功能设计，重造一个Vue出来。

注意！并不是Vue的源码解析，也不是Vue官方版本的minimal。完全是我个人根据Vue的功能，凭空捏的一个变种Vue。做这个的目的只是为了提升自己的框架设计能力！

花了4、5个小时的时间，一步一步捏出来，最后实现的功能大概有这些：

功能点说明
data支持响应式数据
watch监听响应式数据的变化
methods支持方法
components支持添加子组件
props子组件支持props
mounted组件加载
destroyed组件卸载
render使用渲染函数渲染组件，支持子组件，html元素
style支持style内联样式
事件处理支持事件监听
局部重绘只有状态变更的组件才会重绘
虚拟dom组件树转换成了VNode

没有列出的功能都是没有实现的，包括但不限于模版语法、diff算法、计算属性、条件/列表渲染、组件slot、$emit、filter、mixin、全局方法等等。（内容有点太多了，这些都可以慢慢加，并不是重点）。

总结下来就是：Vue的核心设计包括响应式、组件化、虚拟dom、diff算法、模版语法。本次demo实现了响应式、组件化、虚拟dom。

最终的用法和Vue一样：

new Vue({

});

实现步骤

搭出一个Vue架子

0. 准备工作

先创建一个React项目，用CRA也好，用其他模版也好。项目创建好后，在源码目录下创建一个libs/vue.tsx。

1. Vue设置选项

先想好我们要支持哪些功能，然后在vue.tsx中写好Vue的设置选项的类型VueOptions。这样我们new Vue({/* VueOptions */})的时候就能看到这些选项了。

export type VueOptions = {
  el?: HTMLElement | null;
  components?: Record<string, VueOptions>;
  props?: string[];
  data?: () => Record<string, any>;
  watch?: Record<string, (newVal: any, oldVal: any) => void>;
  methods?: Record<string, any>;
  mounted?: () => void;
  destroyed?: () => void;
  render: (h: (tag: string, opts: any, children?: any[]) => VNode) => VNode;
};

2. Vue的class初始化

开始写Vue的class实现。Vue实例需要从VueOptions里读取配置，并初始化，存储为class变量。

export default class Vue {
  _options: VueOptions; // 选项
  el?: HTMLElement | null; // 父级element元素
  components: Record<string, VueOptions>; // 声明的组件
  _componentInstances: Record<string, Vue>; // 使用到的组件实例
  data: Record<string, any>; // 数据
  methods: Record<string, any>; // 数据
  props: Record<string, any>; // 数据
  watchers: Record<string, (newVal: any, oldVal: any) => void>; // 监听器
  mounted: () => void; // 挂载完成回调
  destroyed: () => void; // 销毁回调
  render: (h: (tag: string, opts: any, children?: any[]) => VNode) => VNode; // 渲染函数

  constructor(options: VueOptions) {
    this._options = options;
    // el
    this.el = options.el;
    // components
    this.components = options.components || {};
    // data
    this.data = options.data?.() || {};
    // method
    this.methods = options.methods || {};
    // prop
    this.props = {}
    for (const key of (options.props || [])) {
      this.props[key] = undefined;
    }
    // watch
    this.watchers = options.watch || {};
    // lifecycle
    this.mounted = options.mounted || (() => {});
    this.destroyed = options.destroyed || (() => {});

    this.render = options.render;

    this._init();
  }
  _init() {
    this.mounted();
    // 首次渲染
    this._doRender.call(this);
  }
  ...
}

监听数据变化

数据都获取到了，开始搓Vue最重要的一个功能：响应式。响应式贯穿了Vue很多的功能，不过我们本次demo要实现的功能简化版的：监听data或props的数据变化，并且在变化时，触发watcher和重新渲染。Vue2和Vue3略有不同，Vue2使用Object.defineProperty实现，Vue3使用Proxy实现。我们这次demo直接用Proxy。

考虑到我们要在watcher、method、生命周期里面都要通过this.xxx能访问到data、methods和props。所以将data、methods和props都加到一个proxy实例上，这个proxy是我们要实现响应式的核心。既要通过这个proxy监听数据变化，也要将watcher、method之类的this指向设置到为proxy上。

这里和正版Vue的实现不一样，正版Vue的proxy为Vue实例本身，我们是新加了一个proxy对象。

export default class Vue {
  ...
  // 去掉data、methods、props
  proxy: Record<string, any>; // 代理对象，用于访问this.data、this.methods、this.props
  ...
  constructor(options: VueOptions) {
    // 也去掉data、methods、props的初始化
    this.proxy = this._proxyThis(
      options.data?.(),
      options.methods,
      options.props
    );
  }
  ...
  // 为当前Vue实例创建一个代理
  _proxyThis(data: Record<string, any> = {}, methods: Record<string, any> = {}, props: string[] = []) {
    const that = this;
    const obj = { ...data, ...methods };
    for (const key of props) {
      obj[key] = undefined;
    }
    const proxy = new Proxy(obj, {
      get: (target, p: string, receiver) => {
        const value = target[p];
        if (typeof value === "function") {
          // 为methods绑定proxy
          return value.bind(proxy);
        }
        return value;
      },
      set: (target, p: string, newValue, receiver) => {
        const oldValue = target[p];
        target[p] = newValue;
        // 触发watcher
        if (p in that.watchers) {
          that.watchers[p].call(proxy, newValue, oldValue);
        }
        // 重新渲染
        that._doRender.call(that);
        return true;
      },
    });
    return proxy;
  }
}

代码中，我们为methods和watchers重新设置this指向proxy。同时在setprops和data时，触发watcher和重新渲染。

渲染一：生成虚拟dom

接下来就是渲染过程。本次demo中我们没有使用模版语法，直接使用的是渲染函数。属实是简化了许多。

正版Vue的模版语法，是有解析模版html转为AST -> AST转为调用渲染函数 -> 渲染函数生成虚拟dom -> 虚拟dom更新真实dom。

渲染函数即在每个vue组件中，都实现render函数来告知vue本组件是什么结构。render函数提供一个h方法来创建虚拟dom。

所以我们需要通过我们提供的h方法，将本组件中的虚拟dom节点都记录下来。h方法我们保持和正版vue相同的三个参数：type-节点类型，props-节点属性，children-该节点的子节点数组。节点包含原生dom元素和已注册的vue组件，如果是原生dom元素，那props我们就当作元素的attrs设置上去；如果是组件，那props就作为组件的props传递下去。

先不考虑vue组件，实现一下h方法：

/** 虚拟node节点 */
export type VNode = {
  type: string;
  props: Record<string, any>;
  children: VNode[];
};

export default class Vue {
  ...
  h(type: string, props: any, children: any[] = []): VNode {
    let vnode: VNode;
    if (this.components.hasOwnProperty(type)) {
      // 如果是自定义组件 TODO
    } else {
      // 如果是原生标签
      // 存储为VNode
      vnode = { type, props, children };
      // 添加children
      vnode.children = [];
      for (const child of children) {
        if (typeof child === "string") {
          vnode.children.push({
            type: "text",
            props: { textContent: child },
            children: [],
          });
        } else {
          vnode.children.push(child);
        }
      }
    }
    return vnode;
  }
  ...
}

这样我们就将原生dom元素和纯文本保存为VNode（就是定义的虚拟dom节点）

渲染二：创建真实dom

前面我们在init()和watcher触发时都调用了_doRender()方法，在这里我们就是要执行具体的渲染操作了：将虚拟dom映射为真实dom上。该创建创建该更新更新。不过这个demo也是为了简单，正版vue是需要通过diff算法来判断哪些dom需要创建，哪些dom不需要创建只需要更新，这次的demo先去掉这些，每次渲染都是全部卸载后创建。（日后有时间再补上）

export default class Vue {
  ...
  // 执行渲染（简化版本，没有加diff算法）
  _doRender() {
    if (this.el) {
      const vnodeTree = this.render(this.h.bind(this));
      // 清空el下所有元素
      this.el.innerHTML = "";
      // 重新渲染子树
      this._renderRecursively(vnodeTree, this.el);
    }
  }

  // 递归渲染vnode树
  _renderRecursively(vnode: VNode, parent: HTMLElement) {
    if (vnode.type === "text") {
      // 文本节点
      parent.appendChild(document.createTextNode(vnode.props.textContent));
      return;
    } else if (vnode.type === "Component") {
      // 组件节点 TODO
      return;
    }
    // 其他节点，创建HtmlElement
    const element = document.createElement(vnode.type);
    // 设置属性
    for (const key in vnode.props) {
      if (key === "style") {
        // style
        Object.assign(element.style, vnode.props[key]);
      } else if (key.startsWith("on")) {
        // 事件
        element.addEventListener(key.slice(2).toLowerCase(), vnode.props[key]);
      } else {
        // 其他
        element.setAttribute(key, vnode.props[key]);
      }
    }
    // 递归渲染children
    for (const child of vnode.children) {
      this._renderRecursively(child, element);
    }
    parent.appendChild(element);
  }
  ...
}

_renderRecursively就是递归渲染VNode树。如果是文本虚拟dom，直接append；如果是组件，后面讲；否则作为原生dom元素：创建element，处理一下属性，然后递归处理children，再把处理完的自己加到dom树上。

正版vue处理的属性不止style和on事件。这里简化了。

支持组件化

要支持组件化，主要是两步：一是在提供的h方法里，我们要初始化子组件实例，传递props，调用子组件的render函数获取子组件的VNode，然后把子组件的VNode记录下来，注意要把它和原生元素区分开。二是在实际渲染时，如果有子组件，则递归渲染子组件，然后append到dom树上即可。

1. 创建子组件实例

在原来的h方法里，补充上子组件的实现。注意子组件需要缓存下来，不能每次new一个新的实例。同时为了区分它是个组件类型，创建了一个type: "Component"的虚拟dom。

export default class Vue {
  ...
  h(type: string, props: any, children: any[] = []): VNode {
    let vnode: VNode;
    if (this.components.hasOwnProperty(type)) {
      // 如果是自定义组件
      // 获取组件实例
      let component = this._componentInstances[type];
      if (!component) {
        component = new Vue(this.components[type]);
      }
      this._componentInstances[type] = component;
      // 更新proxy的props
      for (const key in props) {
        component.proxy[key] = props[key];
      }
      // 渲染组件到element
      vnode = {
        type: "Component",
        props,
        _component: component,
        children: [component.render(component.h.bind(component))],
      };
    } else {
      // 如果是原生标签
      ...
    }
    return vnode;
  }
  ...
}

2. 子组件递归渲染

渲染真实dom时，如果有子组件，递归渲染子组件。（需要告知子组件父级真实dom是什么，不然无法append上去）

export default class Vue {
  ...
  // 递归渲染vnode树
  _renderRecursively(vnode: VNode, parent: HTMLElement) {
    if (vnode.type === "text") {
      // 文本节点
      ...
    } else if (vnode.type === "Component") {
      // 组件节点
      if (vnode._component) {
        vnode._component.el = parent;
      }
      // 本身不用append，从第一个child开始
      this._renderRecursively(vnode.children[0], parent);
      return;
    }
    // 其他节点，创建HtmlElement
    ...
  }
  ...
}

至此，Vue的组件化系统也支持完毕了。现在vue组件内可以声明子组件，渲染子组件。当然，子组件也可以继续包含子组件，形成任意级嵌套。属性也能通过props传递下去。最终的demo效果在开篇就已经展示了。

源码

完整的“Vue”的源码如下：

// src/libs/vue.tsx
/** Vue设置选项（同Vue2） */
export type VueOptions = {
  el?: HTMLElement | null;
  components?: Record<string, VueOptions>;
  props?: string[];
  data?: () => Record<string, any>;
  watch?: Record<string, (newVal: any, oldVal: any) => void>;
  methods?: Record<string, any>;
  mounted?: () => void;
  destroyed?: () => void;
  render: (h: (tag: string, opts: any, children?: any[]) => VNode) => VNode;

  [key: string]: any;
};

/** 虚拟node节点 */
export type VNode = {
  type: string;
  props: Record<string, any>;
  children: VNode[];
  _component?: Vue;
};

/** Vue对象 */
export default class Vue {
  _options: VueOptions; // 选项
  el?: HTMLElement | null; // 父级element元素
  components: Record<string, VueOptions>; // 声明的组件
  _componentInstances: Record<string, Vue>; // 使用到的组件实例
  proxy: Record<string, any>; // 代理对象，用于访问this.data、this.methods、this.props
  watchers: Record<string, (newVal: any, oldVal: any) => void>; // 监听器
  mounted: () => void; // 挂载完成回调
  destroyed: () => void; // 销毁回调
  render: (h: (tag: string, opts: any, children?: any[]) => VNode) => VNode; // 渲染函数

  constructor(options: VueOptions) {
    this._options = options;
    // el
    this.el = options.el;
    // components
    this.components = options.components || {};
    this._componentInstances = {};
    // data
    this.proxy = this._proxyThis(
      options.data?.(),
      options.methods,
      options.props
    );
    // watch
    this.watchers = options.watch || {};
    // lifecycle
    this.mounted = options.mounted || (() => {});
    this.destroyed = options.destroyed || (() => {});

    this.render = options.render.bind(this.proxy);

    this._init();
  }

  _init() {
    this.mounted();
    // 首次渲染
    this._doRender.call(this);
  }

  // 为当前Vue实例创建一个代理
  _proxyThis(
    data: Record<string, any> = {},
    methods: Record<string, any> = {},
    props: string[] = []
  ) {
    const that = this;
    const obj = { ...data, ...methods };
    for (const key of props) {
      obj[key] = undefined;
    }
    const proxy = new Proxy(obj, {
      get: (target, p: string, receiver) => {
        const value = target[p];
        if (typeof value === "function") {
          // 为methods绑定proxy
          return value.bind(proxy);
        }
        return value;
      },
      set: (target, p: string, newValue, receiver) => {
        const oldValue = target[p];
        target[p] = newValue;
        // 触发watcher
        if (p in that.watchers) {
          that.watchers[p].call(proxy, newValue, oldValue);
        }
        // 重新渲染
        that._doRender.call(that);
        return true;
      },
    });
    return proxy;
  }

  // 执行渲染（简化版本，没有加diff算法）
  _doRender() {
    if (this.el) {
      const vnodeTree = this.render(this.h.bind(this));
      // 清空el下所有元素
      this.el.innerHTML = "";
      // 重新渲染子树
      this._renderRecursively(vnodeTree, this.el);
    }
  }

  // 递归渲染vnode树
  _renderRecursively(vnode: VNode, parent: HTMLElement) {
    if (vnode.type === "text") {
      // 文本节点
      parent.appendChild(document.createTextNode(vnode.props.textContent));
      return;
    } else if (vnode.type === "Component") {
      // 组件节点
      if (vnode._component) {
        vnode._component.el = parent;
      }
      // 本身不用append，从第一个child开始
      this._renderRecursively(vnode.children[0], parent);
      return;
    }
    // 其他节点，创建HtmlElement
    const element = document.createElement(vnode.type);
    // 设置属性
    for (const key in vnode.props) {
      if (key === "style") {
        // style
        Object.assign(element.style, vnode.props[key]);
      } else if (key.startsWith("on")) {
        // 事件
        element.addEventListener(key.slice(2).toLowerCase(), vnode.props[key]);
      } else {
        // 其他
        element.setAttribute(key, vnode.props[key]);
      }
    }
    // 递归渲染children
    for (const child of vnode.children) {
      this._renderRecursively(child, element);
    }
    parent.appendChild(element);
  }

  h(type: string, props: any, children: any[] = []): VNode {
    let vnode: VNode;

    if (this.components.hasOwnProperty(type)) {
      // 如果是自定义组件
      // 获取组件实例
      let component = this._componentInstances[type];
      if (!component) {
        component = new Vue(this.components[type]);
      }
      this._componentInstances[type] = component;
      // 更新proxy的props
      for (const key in props) {
        component.proxy[key] = props[key];
      }
      // 渲染组件到element
      vnode = {
        type: "Component",
        props,
        _component: component,
        children: [component.render(component.h.bind(component))],
      };
    } else {
      // 如果是原生标签
      // 存储为VNode
      vnode = { type, props, children };
      // 添加children
      vnode.children = [];
      for (const child of children) {
        if (typeof child === "string") {
          vnode.children.push({
            type: "text",
            props: { textContent: child },
            children: [],
          });
        } else {
          vnode.children.push(child);
        }
      }
    }

    return vnode;
  }

  destroy() {
    // 销毁子组件
    for (const key in this._componentInstances) {
      this._componentInstances[key].destroy();
    }
    // 销毁自己
    this.destroyed();
  }
}
